## 第二阶段置信度约束

| 条件 | adjusted_score 上限 |
|------|--------------------|
| 样本数 < 3 | 0.65 |
| 样本数 < 5 | 0.75 |
| 使用替代口径 | 0.70 |
| 仅有替代样本、没有真实标的样本 | 0.60 |
| BD事件库过去365日事件数 < 8 | S2-01 上限 0.70 |
| BD事件库过去365日事件数 < 15 | S2-01 上限 0.80 |
| S1_total < 0.50 且 S1-02 < 0 | S2综合分 ≤ 0.75 |

S2 同时输出两个分数：

```text
S2_raw_total = Σ(S2_i_raw_score × S2_i_weight)
S2_adjusted_total = Σ(S2_i_adjusted_score × S2_i_weight)
S2_total = S2_adjusted_total
```

`S2_raw_total` 只作为参考，不参与阶段判断。正式阶段判断只使用 `S2_adjusted_total`。

## S2 与 S1 的区别

S1 是行情、份额、成交、广度等连续指标，适合日频硬计算。

S2 是产业验证指标，底层依赖 BD、临床、业绩等事件库，因此属于“事件驱动 + 行情转化”的半量化指标。S2 不直接依赖日报文本，而依赖：

- `s2/data/bd_events.csv`
- `s2/data/clinical_events.csv`
- `s2/data/earnings_events.csv`
- `s2/output/s2_item_scores.csv`
- `s2/output/s2_scores.csv`

S2 所有 raw_score 必须经过样本数、替代口径、数据缺失、S1交易状态等置信度约束后，形成 adjusted_score。正式阶段判断只使用 adjusted_score。
